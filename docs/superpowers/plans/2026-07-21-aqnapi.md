# aqnapi Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans or
> subagent-driven-development. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Jednoplikowe CLI `aqnapi.py` (stdlib-only) — klient Napisy24 /
napiprojekt / OpenSubtitles do pobierania i wysyłania napisów, z pełną
dokumentacją i testami.

**Architecture:** Jeden plik z wyraźnymi sekcjami: błędy → warstwa wspólna (HTTP,
hasze, FPS, silnik napisów, krypto, 7z-AES) → konfiguracja → trzej klienci →
CLI (agregujące + per-serwis). Klienci zwracają wspólne dataclasy; CLI nie zna
protokołów.

**Tech Stack:** Python 3.9+, wyłącznie stdlib (`urllib`, `socket`, `ssl`, `json`,
`xml.etree`, `hashlib`, `struct`, `lzma`, `zipfile`, `base64`, `argparse`,
`configparser`, `subprocess`, `unittest`).

## Global Constraints
- Python 3.9+, **tylko biblioteka standardowa** (zero pip).
- Pobrane napisy: zawsze **SRT, UTF-8 z BOM, końce LF**.
- napiprojekt upload: własny **AES-256 + zapis 7z**, hasło `iBlm8NTigvru0Jr0`.
- Hasła nigdy w logach.
- Komunikaty użytkownika po polsku.

### Task 1: Szkielet, błędy, hasze, FPS
**Files:** Create `aqnapi.py`, `tests/test_aqnapi.py`
**Produces:** `oshash(path)->str(16 lower hex)`, `md5_10mb(path)->str`,
`file_md5(path)->str`, `bytes_md5(bytes)->str`, `fps_from_file(path)->float|None`;
wyjątki `AqError/AuthError/NetworkError/NotFoundError/ServerError/ConfigError`.
- [ ] Testy: OSH na syntetycznym pliku (znany wynik), md5_10mb, fps MKV/AVI/MP4 z
  ręcznie zbudowanych nagłówków (DefaultDuration=40_000_000→25.0; RIFF µs/frame;
  MP4 timescale/stts→23.976).
- [ ] Implementacja hashy (OSH: `<Q` LE, mask 2^64, min 128KiB) i `fps_from_file`
  (MKV EBML vint + DefaultDuration; AVI offset32; MP4 ISO BMFF `hdlr='vide'`+
  `mdhd`+`stts`, obsługa `size==1`). Bramka zaufania `22<fps<32` w warstwie CLI.
- [ ] Uruchom testy → PASS.

### Task 3: AES-256 + KDF 7z + zapis 7z
**Produces:** `class AES` (`encrypt_block`), `aes_cbc_encrypt(key,iv,data)`,
`sevenzip_key(password_utf16le, salt, cycles_pow=19)`, `write_7z_aes(name,data)->bytes`.
- [ ] Test: AES-256 wektor FIPS-197 (klucz `000102...1f`, blok `00112233...ff` →
  `8ea2b7ca516745bfeafc49904b496089`).
- [ ] Test: `write_7z_aes` → zapis do pliku → `7z x -p<hasło>` → bajty równe
  wejściu (skip gdy brak binarki `7z`; `shutil.which`).
- [ ] Implementacja AES (S-box, key expansion, CBC), KDF (jeden SHA-256 karmiony
  2^19× `salt+pw+counter_u64LE`), kontener 7z (sygnatura, StartHeader z CRC,
  coder `06F10701`, props firstByte `0x53`+`0x0F`+IV, SubStreamsInfo z CRC32).
- [ ] PASS + weryfikacja binarką.
